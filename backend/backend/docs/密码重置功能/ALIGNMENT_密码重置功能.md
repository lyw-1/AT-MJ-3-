# 密码重置功能需求对齐文档

## 原始需求
- 实现密码重置相关接口
- 实现密码重置业务逻辑
- 实现密码重置缓存设计

## 边界确认
- **功能范围**：实现用户忘记密码时的密码重置流程，包括发送重置验证码、验证重置码、通过重置码重置密码
- **不包含**：邮件发送具体实现（仅设计接口）、短信发送具体实现、前端页面开发

## 需求理解
基于现有项目架构，密码重置功能是用户权限管理模块的重要组成部分。该功能允许用户在忘记密码时，通过用户名/邮箱/手机号获取重置验证码，然后使用验证码设置新密码。

## 项目理解
- 项目使用Spring Boot 2.7作为基础框架
- 使用Spring Security进行认证和授权
- 使用Redis进行缓存管理
- 采用分层架构：Controller → Service → Mapper → Model
- 已有的用户实体包含username、email、phone等字段
- 已有密码重置相关的数据传输对象

## 疑问澄清
- 验证码有效期：设置为15分钟
- 发送频率限制：每分钟最多发送1次
- 24小时内发送次数限制：最多10次
- 验证码位数：6位数字
