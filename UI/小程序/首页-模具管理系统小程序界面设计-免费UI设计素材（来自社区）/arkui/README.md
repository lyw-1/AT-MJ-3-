
## 目录详细说明

### `entry/src/main`

#### `ets/`

- **components/**  
  存放可复用的组件，组件数据来源于设计稿中的组件，组件集。

- **pages/**  
  存放页面组件，每个文件代表一个页面。

- **entity/**  
  存放类型定义，例如组件引用传参中参数的接口定义。

- **router/**  
  存放路由数据，用在例如页面跳转场景下。

- **style/**  
  存放样式数据，样式数据来源于设计稿。

- **utils/**  
  存放工具类以及屏幕适配类。

- **variables/**  
  存储所有变量信息。

- **variableslayer/**  
  存储图层应用变量模式信息，可以获取图层应用变量值、更新图层变量集模式。

- **variablesset/**  
  存储所有变量集信息。

#### `resources/`
存放静态资源（如图片、字体，基本数据类型），例如样式数据中填充类型数据存在 `color.json` 中。

### `README.md`
项目的说明文档，介绍目录结构，代码结构。

## 代码说明
1. 代码规范遵循组件+实例，生成的时候识别到图层是实例，会引用相关组件，达到代码简洁以及提升代码可用性。

2. 在生成的代码中，会遇到属性是变量，样式模式。样式，变量规则可以参考设计稿规则。

3. 在设计稿转换为代码过程中，遇到复杂的矢量图层或者复杂的图层表现形式中，例如图片采用了裁剪模式，我们会将该图层转化为图片。

4. 我们支持复杂的原型交互，例如页面跳转，鼠标事件，点击事件，以及滚动方向支持。其他原型交互进一步支持中。

5. 如果遇到生成的代码编译错误，请联系产品客服，我们会提供远程解决方案。
