# 文案表达优化指南

## 优化目标

通过系统性的文案优化，提升用户体验，确保信息传达清晰、准确、友好，符合品牌调性。

## 一、文案优化原则

### 1.1 基本原则

1. **清晰简洁**
   - 使用简单易懂的词汇
   - 避免专业术语和缩写
   - 每句话不超过20个字
   - 优先表达核心信息

2. **准确具体**
   - 明确传达操作结果
   - 提供具体的时间和数量
   - 避免模糊的表达（如"很快"、"很多"）
   - 使用精确的数字和单位

3. **友好礼貌**
   - 使用积极的语言
   - 避免指责用户
   - 提供解决方案而非仅指出问题
   - 使用"请"、"建议"等礼貌用语

4. **一致性**
   - 统一术语和表达方式
   - 保持相同功能使用相同文案
   - 遵循品牌调性
   - 保持与设计稿一致

## 三、文案使用规范

### 3.1 标点使用

1. **标题标点**
   - 使用句号"。"或"？"
   - 不使用省略号"..."
   - 标点后加空格

2. **逗号使用**
   - 中文逗号后加空格
   - 列表项之间用逗号分隔
   - 最后一个列表项后用"和"或"或"

### 3.2 数字格式

1. **数字表达**
   - 使用阿拉伯数字
   - 大数字使用千分位分隔（1,000）
   - 小数保留两位小数（12.5%）

2. **单位使用**
   - 使用标准单位（%、个、次、分钟）
   - 单位与数字之间加空格
   - 避免使用不规范的单位

### 3.3 大小写

1. **专有名词**
   - 品牌名、产品名：首字母大写
   - 功能模块：首字母大写
   - 按钮文字：首字母大写

2. **普通名词**
   - 使用小写开头
   - 句子开头小写
   - 避免全大写

## 五、品牌调性

### 5.1 品牌调性

1. **品牌色**
   - 主色：#74CA00（绿色）
   - 成功色：#10B981
   - 警告色：#F59E0B
   - 错误色：#EF4444
   - 信息色：#6B7280

2. **品牌语调**
   - 专业但不生硬
   - 友好但不随意
   - 清晰但不啰嗦
   - 积极但不夸大

3. **避免使用的表达**
   - "非常"（避免过度强调）
   - "超级"（避免夸大）
   - "绝对"（避免绝对化）
   - "马上"（避免催促）

## 六、实施建议

### 6.1 立即实施

1. **创建文案字典**
   ```javascript
   // constants/text.js
   module.exports = {
     SUCCESS: '✓ 操作完成',
     ERROR_NETWORK: '网络连接失败，请检查网络后重试',
     ERROR_EMPTY: '暂无数据，快去创建吧',
     CONFIRM_DELETE: '确定要删除此项目吗？',
     LOADING: '加载中...'
   };
   ```

2. **使用文案工具函数**
   ```javascript
   // utils/text.js
   function formatNumber(num) {
     return num.toString().replace(/\B(?=(\d{3})+(?!\d))/g, ',');
   }
   
   function formatTime(minutes) {
     if (minutes < 60) {
       return `${minutes}分钟前`;
     } else if (minutes < 1440) {
       const hours = Math.floor(minutes / 60);
       const mins = minutes % 60;
       return `${hours}小时${mins}分钟前`;
     } else {
       const days = Math.floor(minutes / 1440);
       return `${days}天前`;
     }
   }
   ```

3. **逐步替换现有文案**
   - 优先替换高频文案
   - 保持一致性
   - 测试用户体验

### 6.2 测试验证

1. **A/B测试**
   - 对比不同文案的效果
   - 收集用户反馈
   - 选择最优方案

2. **用户测试**
   - 邀请真实用户测试
   - 观察用户理解
   - 收集改进建议

3. **可用性测试**
   - 测试文案可读性
   - 验证不同设备显示
   - 检查多语言支持

### 6.3 持续优化

1. **定期审查**
   - 每月审查文案使用情况
   - 根据反馈调整优化
   - 更新文案字典

2. **用户反馈**
   - 收集用户对文案的反馈
   - 分析常见误解
   - 持续改进表达

3. **行业对标**
   - 参考优秀产品文案
   - 学习最佳实践
   - 保持行业先进性

## 七、注意事项

### 7.1 本地化

1. **多语言支持**
   - 使用i18n方案
   - 分离文案和代码
   - 支持中英文切换

2. **文化适配**
   - 注意文化差异
   - 避免文化敏感词汇
   - 使用中性表达

### 7.2 法律合规

1. **隐私保护**
   - 避免收集不必要信息
   - 明确隐私政策
   - 遵循相关法规

2. **用户协议**
   - 明确服务条款
   - 清晰说明用户权利
   - 提供申诉渠道

## 八、总结

通过系统性的文案优化，可以实现：

1. **用户体验提升** - 更清晰、准确、友好的信息传达
2. **品牌一致性** - 统一的品牌调性和表达方式
3. **错误减少** - 减少用户误解和操作错误
4. **专业性提升** - 更专业但不生硬的沟通方式
5. **可维护性** - 集中管理的文案字典和工具函数

建议按照本指南逐步实施文案优化，并在每个阶段进行用户测试和反馈收集，持续改进用户体验。
